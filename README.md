# Постановка задачи
Необходимо по фотографии одного или нескольких предметов и многоугольника, который нарисован черным маркером на белом листе A4, определить, могут ли каким-либо способом данные предметы одновременно, но не перекрывая друг друга, поместиться в этот многоугольник. 
Предметы и лист с многоугольником располагаются на горизонтальной поверхности. Все предметы, которые могут оказаться на фотографии, заранее известны.
## О программе
- В основном файле программы `intelligent_placer.py` содержится функция `check_image` на вход она принимает путь до изображения с предметами на фоне и массив координат вершин многоугольника. Функция возвращает True, если предметы можно вместить в многоугольник, иначе - False.
- В случае, если запущен скрипт `intelligent_placer.py`, то ему в качестве параметров передаётся путь к изображению и путь к файлу с описанием многоугольника. Программа выводит в стандартный поток ввода результат выполнения функции check_image: True или False.

## Требования
### Общие требования к изображениям
* Все изображения в .jpg формате
* Угол между направлением камеры и перпендикуляром к поверхности должен быть не более *15&deg;*
* Изображения сделаны в дневном освещении, объекты на фотографии имеют чёткую границу, отсутствуют засвеченные области и тени высокой интенсивности
* Изображения цветные, без цветовой коррекции и наложения фильтров
* Объекты  должны быть размещены на светлой поверхности из [файла](images/background.jpg)

### Требования к многоугольнику
Многоугольник должен быть выпуклым и замкнутым.
#### Формат многоугольника
Мноугольник задаётся как массив пар координат. Порядок пар задаёт стороны - вершина соединяется с предыдущей и следующей.
Последняя и первая вершина также считаются соединнёными.

Масштаб координат соответствует масштабу пикселей. Например, отрезок `(10, 20)-(100, 40)` соответствует отрезку из пикселя с координатой `(10, 20)` в пиксель с координатой `(100, 40)`.

Файл, задающий n-угольник содержит n-строк, в каждой из который содержится 2 числа - координаты вершины. Порядок строк соответсвует порядку вершин в массиве.
Например, такой файл задаёт квадрат 100x100:
```
100 100
100 200
200 200
200 100
```

### Требования к объектам
* На фото находятся объекты только из множества рассматриваемых примитивов
* Один объект присутствовать на фото только один раз
* Объекты не должны накладываться друг на друга
* Объекты должны целиком помещаться на фото
